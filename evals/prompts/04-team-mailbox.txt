Use the workflow tool to create a small, bounded mailbox team. Keep the mailbox flow simple so the workflow completes quickly: spawn an architect and one worker with mailbox:true, connect them with mailbox.connect(), then use supervisor mailbox.send(worker.id, ...) to seed the Task data model contract (fields: id, title, done) to the worker. The worker should read the supervisor-seeded contract, produce a short implementation plan, send one acknowledgement to the architect via mailbox_send, and finish. The architect should not wait indefinitely; it may summarize its design and any acknowledgement if received. Do not create a multi-step cross-review loop. Await all spawned handles before the phase returns. End with a synthesis phase that confirms the mailbox transcript exists and the Task contract reached the worker.
