Use the workflow tool to create a small, bounded mailbox team. Keep the mailbox flow simple so the workflow completes quickly: spawn an architect and one worker with mailbox:true, connect them with mailbox.connect(), and have the architect send a Task data model contract (fields: id, title, done) to the worker via mailbox_send. The worker should read the contract, produce a short implementation plan, send one acknowledgement back, and finish. Do not create a multi-step cross-review loop and do not pause indefinitely. Await all spawned handles before the phase returns. End with a synthesis phase that confirms the mailbox transcript exists and the Task contract flowed from architect to worker.
